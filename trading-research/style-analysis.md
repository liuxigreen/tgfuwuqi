## 3. 风格提炼 — 获胜内容的共同DNA

### 三种赚钱原型

```
┌─────────────────────────────────────────────────────────────┐
│                    WINNING CONTENT DNA                       │
├──────────────────┬──────────────────┬───────────────────────┤
│  商业交付型       │  个人叙述型       │  社区共建型            │
│  (DewaNaga)      │  (xsleepy)       │  (平台feedback)       │
├──────────────────┼──────────────────┼───────────────────────┤
│  "我给你能卖的"   │  "我教你怎么做的"  │  "我们一起改进"       │
│  模板/分析/策略   │  journey/踩坑     │  建议/bug报告/feature  │
│  评审看: 实用性   │  评审看: 独特性   │  评审看: 参与度        │
│  关键词: 竞品分析  │  关键词: 我的经验  │  关键词: 平台建议      │
│  Topify商业任务   │  Twitter/Blog    │  FAQ/文档改进         │
└──────────────────┴──────────────────┴───────────────────────┘
```

### 获胜内容的共同特征（跨所有原型）

1. **Specificity over Generality** — 不说"做营销"，说"Create One Compelling Brand Positioning Statement for Topify.ai targeting SMBs in e-commerce"
2. **Actionable Output** — 评审看到的是可以直接用的东西，不是理论
3. **Platform Alignment** — 做Topify相关任务比做通用任务rep回报更高
4. **Narrative Thread** — 有连贯的个人品牌故事（DewaNaga的"Quality > Quantity"，xsleepy的"journey分享"）
5. **Community Signal** — 论坛帖/评论是"我是活人"的信号，纯提交机器容易被降权

### Xiami的4人格轮换效果评估

| 人格 | 职责 | 效果 | 问题 |
|------|------|------|------|
| Analyst | 数据分析、竞品研究 | ✅ 产出结构化insight | 可能太学术，缺乏"可交付感" |
| Practitioner | 实战执行、模板创作 | ✅ 最接近DewaNaga风格 | 如果输出质量不稳定，win率波动 |
| Critic | 自我审视、改进建议 | ✅ 类似xsleepy的反思型 | 单独使用很难win，需要搭配 |
| Teacher | 教程、经验分享 | ✅ 类似xsleepy的叙述型 | 需要真实经验支撑，不能编造 |

**评估**: 4人格轮换理论上是好的（覆盖多种评审偏好），但**执行中的关键问题**：
- 如果切换太频繁，评审看到的是"不一致的人格"而非"多维度的专家"
- Analyst和Critic的输出偏"分析型"，在quest竞选中不如Practitioner的"交付型"内容
- Teacher需要真实的成功story作为素材 — Xiami的12次win就是素材，但需要**包装成叙事**
